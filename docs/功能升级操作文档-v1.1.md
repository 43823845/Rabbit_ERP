## 二、功能操作指南

### 2.1 损益结转凭证状态展示

**位置：** 系统设置 → 期间管理 → 结账页面

**功能：** 结账页面的卡片网格下方新增「损益结转凭证」信息条，实时显示当前年份系统自动生成的损益结转凭证数量。

- 当有已结账期间时，自动统计该年度所有 `[损益结转]` 标记的凭证
- 提示文字："结账时系统自动生成，将当期损益类科目余额结转至「本年利润」"

**业务流程：**
```
录入凭证 → 审核凭证 → 过账凭证 → 期末结账（自动生成结转凭证）→ 结账页面可查看结转凭证统计
```

### 2.2 操作日志系统

**位置：** 系统设置 → 操作日志

#### 日志记录范围

| 操作类型 | 记录时机 |
|----------|----------|
| 创建凭证 | 新增凭证保存时 |
| 删除凭证 | 删除凭证确认后 |
| 审核凭证 | 单条审核成功后 |
| 过账凭证 | 单条过账成功后 |
| 批量审核 | 多选审核完成后 |
| 批量过账 | 多选过账完成后 |
| 期末结账 | 结账操作成功后 |
| 创建模板 | 保存凭证模板时 |
| 删除模板 | 删除凭证模板时 |
| 添加摘要 | 新增常用摘要时 |
| 删除摘要 | 删除常用摘要时 |

#### 日志查看

1. 进入 `系统设置` → 左侧导航选择 `操作日志`
2. 每条日志显示：操作标签、操作目标、详细描述、操作用户、时间戳
3. 支持按日期范围筛选和时间倒序排列
4. 可选择显示最近 30/50/100/200 条
5. 系统最多保留 **500 条**日志，超出后自动裁剪最早的记录

#### 日志数据结构

```typescript
{
  id: number;          // 自增 ID
  userId: number;      // 操作用户 ID
  username: string;    // 操作用户名
  action: string;      // 操作类型（创建凭证、审核、过账、期末结账等）
  target: string;      // 操作目标（凭证字号、科目编码等）
  detail: string;      // 详细描述
  createdAt: string;   // 操作时间（ISO 格式）
}
```

### 2.3 批量审核/过账

**位置：** 凭证管理页面顶部工具栏

#### 批量审核

1. 在凭证列表中勾选多张 **草稿** 状态凭证
2. 点击顶部 **「批量审核(N)」** 按钮（橙色边框，N 为可审核凭证数）
3. 确认对话框后系统批量执行审核

**⚠ 审核分离规则：** 如果当前用户是凭证的制单人，该凭证审核会失败。专业财务规范要求制单和审核必须由不同人员完成。

#### 批量过账

1. 勾选多张 **已审核** 状态凭证
2. 点击顶部 **「批量过账(N)」** 按钮（蓝色边框）
3. 确认后批量执行过账

**操作结果：** 批量操作完成后弹出提示 "成功 X 张，失败 Y 张"。

---

### 2.4 制单审核分离规则

**规则：** 审核凭证时，系统自动检查当前用户是否为该凭证的制单人。

- **制单人 ≠ 审核人** → 审核通过，状态变为「已审核」
- **制单人 = 审核人** → 审核失败，提示 "制单人与审核人不能为同一人，请由其他用户审核"

**适用场景：** 多人协作的会计团队中，由会计录入凭证后，由审计员角色进行审核。

**角色建议：**
```
系统管理员/admin  → 管理用户和系统配置
会计/accountant   → 录入凭证（制单）
审计员/auditor    → 审核凭证
查看者/viewer     → 只读查看
```

---

### 2.5 科目删除引用检查

**位置：** 会计科目页面 → 行操作 → 删除按钮

**变更说明：** 删除科目时，系统会自动检查：

1. **是否被凭证引用：** 检查所有凭证分录中是否使用了该科目编码
2. **是否有子科目：** 检查是否有以该科目为父科目的子科目

**确认对话框：**
- 无引用 → 直接确认删除
- 有引用 → 显示红色警告 "该科目在 N 笔凭证中被引用" 或 "该科目下存在子科目"，提醒用户数据异常风险

### 2.6 凭证模板与摘要库（后端就绪）

> **说明：** 后端 API 已完成，前端模板选择器界面将随后续版本发布。

#### API 接口

| 接口 | 说明 |
|------|------|
| `listVoucherTemplates()` | 获取凭证模板列表 |
| `saveVoucherTemplate(name, entries)` | 保存凭证模板 |
| `deleteVoucherTemplate(id)` | 删除凭证模板 |
| `listVoucherSummaries()` | 获取常用摘要列表 |
| `createVoucherSummary(text, category)` | 添加常用摘要 |
| `deleteVoucherSummary(id)` | 删除常用摘要 |

#### 数据结构

**凭证模板：**
```typescript
{
  id: number;
  name: string;                          // 模板名称，如"每月工资计提"
  entries: Array<{
    summary: string;                     // 分录摘要
    subjectCode: string;                 // 科目编码
    subjectName: string;                 // 科目名称
  }>;
  shareType: 'personal' | 'shared';     // 个人/共享
  createdAt: string;
}
```

**常用摘要：**
```typescript
{
  id: number;
  text: string;      // 摘要内容，如"计提本月工资"
  category: string;  // 分类，如"工资"、"报销"、"收入"
  createdAt: string;
}
```

## 六、注意事项

1. **操作日志上限：** 系统最多保留 500 条日志，超出后自动删除最早记录。建议定期导出备份。
2. **审核分离：** 单用户模式下无法审核自己制作的凭证，需切换至其他用户账号或使用管理员角色。
3. **批量操作：** 批量审核和批量过账是独立操作，不支持将选中的凭证"一键审核+过账"。
4. **科目删除：** 即使系统显示引用警告，用户仍可选择删除，但会导致相关凭证出现孤立科目编码，请谨慎操作。
